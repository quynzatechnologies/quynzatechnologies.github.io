  • 4.1. Encendido y apagado de LEDs

Los LEDs (Light Emitting Diodes) son componentes electrónicos que emiten luz cuando circula corriente eléctrica a través de ellos. En el ámbito de los microcontroladores, como el ESP32, los LEDs son uno de los primeros periféricos que se utilizan para aprender a controlar salidas digitales, ya que permiten visualizar de forma inmediata el resultado de la programación.

  1. Principio de funcionamiento

    Un LED solo se enciende si está correctamente polarizado:

    • Ánodo (positivo) conectado al pin de salida del microcontrolador (GPIO).
    • Cátodo (negativo) conectado a tierra (GND) a través de una resistencia limitadora.

    En el ESP32, algunos módulos incluyen un LED interno (usualmente conectado al GPIO 2) que facilita las pruebas iniciales sin necesidad de cablear un circuito externo.

  2. Control de LEDs en MicroPython

    En MicroPython, el control de LEDs se realiza utilizando el módulo machine, que permite configurar un pin como salida digital y cambiar su estado entre:

    • 1 o True -> Encendido.
    • 0 o False -> Apagado.

    Ejemplo básico: 

    from machine import Pin
import time

led = Pin(2, Pin.OUT)  # Configura el GPIO 2 como salida
led.value(1)           # Enciende el LED
time.sleep(1)          # Espera 1 segundo
led.value(0)           # Apaga el LED

  3. Temporización y parpadeo

    Para generar parpadeos o secuencias, se emplea el módulo time y su función sleep(segundos), que pausa la ejecución del programa por el tiempo indicado.

    El parpadeo puede implementarse:

    • Una sola vez (control secuencial).
    • De forma repetitiva con estructuras como while True o bucles for.

  4. Aplicaciones prácticas

    Aunque encender un LED pueda parecer básico, este concepto se utiliza en múltiples aplicaciones:

    • Indicadores de estado (sistema encendido, alerta, error).
    • Señales visuales en sistemas de seguridad o control industrial.
    • Comunicación visual simple en prototipos IoT.
