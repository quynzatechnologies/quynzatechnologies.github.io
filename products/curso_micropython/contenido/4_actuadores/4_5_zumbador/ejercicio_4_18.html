Ejercicio 4-18: Reproducir melodía en zumbador usando PWM

Objetivo. Aprender a utilizar el zumbador para interpretar una melodía.

Procedimiento.

  1. Conectar el pin GPIO19 del ESP32 al pin positivo del zumbador en el entrenador QMT-020.



  2. Abrir Thonny y crear un nuevo archivo con extensión .py.
  3. Escribir el código mostrado a continuación, que define las notas musicales y reproduce una melodía.
  4. Ejecutar el programa y escuchar la melodía emitida por el zumbador. 

    import machine
import time

# Crea un objeto PIN que representa el pin 19 Pin y lo deja en modo salida 
buzzer = machine.Pin(19, machine.Pin.OUT)

# Define una función tono que toma como entrada un objeto Pin
# que representa el zumbador, una frecuencia in Hz, y una duración en milisegundos
def tono(pin, frequency, duration):
    pin.freq(frequency) # Set the frequency
    pin.duty(512) # Set the duty cycle
    time.sleep_ms(duration) # Pause for the duration in milliseconds
    pin.duty(0) # Set the duty cycle to 0 to stop the tone

# Define el pin GPIO pin que se conecta al zumbador
buzzer = machine.PWM(machine.Pin(19))

# Define las frecuencias de las notas en Hz
C5 = 523
D5 = 587
E5 = 659
F5 = 698
G5 = 784
A5 = 880
B5 = 988

# Define las duraciones de las notas en milisegundos
quarter_note = 250
half_note = 300
whole_note = 1000
    
# Define la melodia como una lista de tuples (nota, duración)
melody = [
        (E5, quarter_note),
        (E5, quarter_note),
        (F5, quarter_note),
        (G5, half_note),
        (G5, quarter_note),
        (F5, quarter_note),
        (E5, quarter_note),
        (D5, half_note),
        (C5, quarter_note),
        (C5, quarter_note),
        (D5, quarter_note),
        (E5, half_note),
        (E5, quarter_note),
        (D5, quarter_note),
        (D5, half_note),
        (E5, quarter_note),
        (E5, quarter_note),
        (F5, quarter_note),
        (G5, half_note),
        (G5, quarter_note),
        (F5, quarter_note),
        (E5, quarter_note),
        (D5, half_note),
        (C5, quarter_note),
        (C5, quarter_note),
        (D5, quarter_note),
        (E5, half_note),
        (D5, quarter_note),
        (C5, quarter_note),
        (C5, half_note)]
    
    # Play the melody
for note in melody:
    tono(buzzer, note[0], note[1])
    time.sleep_ms(50)

Actividades sugeridas.

  • Analizar la estructura de datos tupla utilizada en la melodía.
  • Aumentar los intervalos de tiempo para hacer la melodía más lenta.
  • Disminuir los intervalos para aumentar la velocidad.
  • Crear una melodía propia cambiando la secuencia de notas y duraciones.
  • Guardar el programa en la carpeta local del computador.
  • Guardar el programa directamente en el microcontrolador y ejecutarlo sin conexión al PC.
