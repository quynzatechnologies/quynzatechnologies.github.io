  • 5.3. Lectura de pulsadores

Los pulsadores son dispositivos de entrada digital que permiten al usuario interactuar con un sistema electrónico enviando señales binarias (0 o 1). Al presionar el pulsador, el circuito se cierra y el pin del microcontrolador recibe un nivel lógico alto (HIGH); al soltarlo, el circuito se abre y se recibe un nivel lógico bajo (LOW).

  1. Conexión y funcionamiento

    En el ESP32, los pulsadores se conectan a pines configurados como entradas digitales (Pin.IN).

    • Estado presionado: el pin recibe un voltaje (normalmente 3.3 V).
    • Estado liberado: el pin queda conectado a tierra (0 V) mediante una resistencia de pull-down (interna o externa).

    En el entrenador QMT-020, los pulsadores suelen estar cableados a pines específicos (por ejemplo, GPIO14) con resistencias integradas para evitar lecturas erróneas.

  2. Lectura en MicroPython

    1. Configurar el pin como entrada (Pin.IN).
    2. Leer su valor con .value() dentro de un ciclo.
    3. Ejecutar acciones en función del valor recibido.

    Ejemplo básico: 

    from machine import Pin
import time

pulsador = Pin(14, Pin.IN)
led = Pin(26, Pin.OUT)

while True:
    if pulsador.value() == 1:
        led.value(1)
    else:
        led.value(0)
    time.sleep(0.1)

  3. Modo básico y modo toggle

    • Modo básico: el LED se enciende mientras el pulsador está presionado y se apaga al soltarlo.
    • Modo toggle: cada pulsación cambia el estado del LED, manteniéndose encendido o apagado hasta la siguiente pulsación. Este modo requiere detectar el flanco de cambio en la señal.

  4. Aplicaciones

    • Encendido/apagado de dispositivos manualmente.
    • Selección de opciones en menús interactivos.
    • Activación de funciones en robots o sistemas de control.
    • Integración con sensores para activar alarmas.
